## Run protocol

1. Download the cleaned LongMemEval corpus into `benchmarks/longmemeval/data/` (gitignored).
2. `cargo build --release -p stoa-bench`
3. `target/release/stoa-bench longmemeval --data-path benchmarks/longmemeval/data/`
4. Output published here on every backend swap; per [ROADMAP.md M4 exit criteria](../../ROADMAP.md) recall@k must regress no more than 5pp on a backend swap before merge.

## CI gate

`stoa-bench longmemeval --dry-run` is the M4 CI gate (no dataset required); a real run is gated to nightly because the dataset + scorer cost is non-trivial.

## Gameability notes

Per `benchmarks/longmemeval/README.md`: backends that game the knowledge-update category by overfitting to LongMemEval question style must also pass MemoryAgentBench FactConsolidation without per-benchmark prompt tuning. Long-context backbones (Claude 4.x with 1M context) score competitively without any memory system; the headline number is the **delta vs the no-memory baseline at the same backbone**, not absolute recall.
